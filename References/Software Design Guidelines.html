Class Design

Follow the Design Guidelines for Developing Class Libraries when designing classes, with the following exceptions:

1) Add the "_" prefix for type member fields.

 

User Interface

 

Follow the Windows User Experience Interaction Guidelines for general direction when building user interfaces for custom Windows applications.

1)      DevExpress controls should be used in most cases. Some customers do not want/wish to use them though, in which case standard Windows controls should be used.

2)      In place of the Segoe UI use the Tahoma font

3)      Within editor controls (lists, textboxes, dropdowns, etc) use Tahoma 8.25pt font

4)      Command buttons should be sized as 75px X 23px, unless it adds clarity to the view to increase the width of the button to accommodate the button text

5)      Action panels (button panels at the bottom of dialogs, etc):

a.      Should be docked to the bottom of the containing view

b.      Should have a height of 45px

c.       Controls contained within the panel should be centered vertically

6)      Message panel (panel showing a info/warning/error icon and message within the Action panel)

a.      Should be docked left within the Action panel

b.      Should have the message text label sized at 11pt, normal weight

 

Handlers

The purpose of a handler is to join a View and a Model, provide inter-handler and controller messaging, and manage menus.

1)      Do not initialize menu fields (member variables) to null (Nothing in Visual Basic). The reason is that the menu field is managed by the superclass ViewHandlerBase

 

Models

The purpose of a model is to provide domain specific data and state for the View to consume.

 

Business Entities

The purpose of a business entity is to provide an object oriented way of looking at the problem domain. The p3 Technologies framework provides entities that are modeled after the CSLA business objects amongst others.

1)      Entities are not instantiated directly under normal use; each entity implements a factory to create instances of individual entities.

2)      Entity Ctor’s are always private
